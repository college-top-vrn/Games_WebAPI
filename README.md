## Модель Competition

Что хранится в **Competition**:

```json
{
  "id": 1,
  "name": "Марафон",
  "date": "2026-05-15T00:00:00",
  "location": "Москва",
  "sportType": "Бег",
  "itDeleted": false
}
```

Поля:
- `Id` – идентификатор соревнования (обязательное поле).
- `Name` – название соревнования (обязательное поле).
- `Date` – дата проведения (обязательное поле).
- `Location` – место проведения (обязательное поле).
- `SportType` – вид спорта (обязательное поле).
- `ItDeleted` – логическое поле, пометка о мягком удалении (true = считается удалённым).

***

## Модель Result

Что хранится в **Result**:

```json
{
  "id": 1,
  "competitionId": 1, 
  "participantName": "Иван Иванов",
  "place": 1,
  "score": 123.45,
  "itDeleted": false
}
```

Поля:
- `Id` – идентификатор результата (обязательное поле).
- `CompetitionId` – ссылка на `Competition.Id` (обязательное поле).
- `ParticipantName` – имя/ФИО участника (обязательное поле).
- `Place` – занятое место (обязательное поле, > 0).
- `Score` – очки или числовой результат (необязательное, может быть `null`).
- `ItDeleted` – логическое поле для мягкого удаления (true = считается удалённым).

## Competitions API (`/api/competitions`)

### GET `/api/competitions/`

Получить список всех **активных** соревнований (`ItDeleted == false`).

- Тело запроса: отсутствует.

**Ответы:**

- `200 OK` — если есть хотя бы одно активное соревнование.  
  Пример ответа:
  ```json
  [
    {
      "id": 1,
      "name": "Московский марафон",
      "date": "2026-05-15T00:00:00",
      "location": "Москва",
      "sportType": "Бег",
      "itDeleted": false
    },
    {
      "id": 2,
      "name": "Чемпионат города по шахматам",
      "date": "2026-06-01T10:00:00",
      "location": "Москва",
      "sportType": "Шахматы",
      "itDeleted": false
    }
  ]
  ```

- `204 No Content` — если активных соревнований нет (либо список пустой, либо всё с `ItDeleted == true`).

***

### GET `/api/competitions/{id}`

Получить одно соревнование по `id`, только если оно **не** помечено как удалённое.

- Параметры пути:
  - `id` (int) — идентификатор соревнования.

**Ответы:**

- `200 OK` — если соревнование найдено и `ItDeleted == false`.  
  Пример ответа:
  ```json
  {
    "id": 1,
    "name": "Московский марафон",
    "date": "2026-05-15T00:00:00",
    "location": "Москва",
    "sportType": "Бег",
    "itDeleted": false
  }
  ```

- `404 Not Found` — если соревнование с таким `id` не найдено или помечено как удалённое (`ItDeleted == true`).

***

### POST `/api/competitions/`

Создать новое соревнование.

- Тело запроса (JSON), модель `Competition`:

```json
{
  "id": 3,
  "name": "Кубок города по плаванию",
  "date": "2026-07-10T09:00:00",
  "location": "Воронеж",
  "sportType": "Плавание",
  "itDeleted": false
}
```

**Условия/валидация:**
- Проверяется, что **нет** соревнования с таким же `Id` в списке.
- Остальные поля не проверяются на стороне POST-контроллера (кроме того, что они придут из модели; атрибуты `[Required]` повлияют при валидации модели, если она включена).

**Ответы:**

- `201 Created` — соревнование успешно добавлено.  
  Заголовок `Location: /api/competitions/{id}`.  
  Пример ответа:
  ```json
  {
    "id": 3,
    "name": "Кубок города по плаванию",
    "date": "2026-07-10T09:00:00",
    "location": "Воронеж",
    "sportType": "Плавание",
    "itDeleted": false
  }
  ```

- `400 Bad Request` — если соревнование с таким `Id` уже существует.  
  Пример ответа:
  ```json
  "Соревнование с таким ID уже существует."
  ```

***

### PUT `/api/competitions/{id}`

Обновить существующее соревнование по `id`.

- Параметры пути:
  - `id` (int) — идентификатор обновляемого соревнования.
- Тело запроса (JSON), модель `Competition` (используется как источник новых значений):

```json
{
  "name": "Московский марафон 2026",
  "date": "2026-05-20T00:00:00",
  "location": "Москва",
  "sportType": "Бег",
  "itDeleted": false
}
```

**Логика:**
- В списке `competitions` ищется элемент с указанным `id`.
- Если не найден, возвращается 404.
- Проверяется:
  - `Name` не пустой/не пробельный.
  - `Location` не пустой.
  - `SportType` не пустой.
  - `Date != DateTime.MinValue`.
- Если проверка не проходит — 400.
- Если всё ок — создаётся новый объект `Competition` (с тем же `Id`), остальные поля берутся из `updatedCompetition`, сохраняется в файл, возвращается обновлённый объект.

**Ответы:**

- `200 OK` — соревнование успешно обновлено.  
  Пример ответа:
  ```json
  {
    "id": 1,
    "name": "Московский марафон 2026",
    "date": "2026-05-20T00:00:00",
    "location": "Москва",
    "sportType": "Бег",
    "itDeleted": false
  }
  ```

- `400 Bad Request` — если одно или несколько обязательных полей не заполнены или `Date == "0001-01-01T00:00:00"`.  
  Пример ответа:
  ```json
  "Name, Location, SportType и Date обязательны."
  ```

- `404 Not Found` — если соревнование с таким `id` не найдено в списке.

***

### DELETE `/api/competitions/{id}`

Мягкое удаление соревнования: не удаляет объект из списка, а меняет `ItDeleted = true`.

- Параметры пути:
  - `id` (int) — идентификатор соревнования.

**Логика:**
- Ищется соревнование по `id` (без фильтрации по `ItDeleted`).
- Если нет такого — 404.
- Если уже `ItDeleted == true` — 410.
- Иначе:
  - `ItDeleted` устанавливается в `true`.
  - Список сохраняется в файл.
  - Возвращается обновлённый объект.

**Ответы:**

- `200 OK` — соревнование помечено как удалённое.  
  Пример ответа:
  ```json
  {
    "id": 1,
    "name": "Московский марафон",
    "date": "2026-05-15T00:00:00",
    "location": "Москва",
    "sportType": "Бег",
    "itDeleted": true
  }
  ```

- `404 Not Found` — соревнование с таким `id` не найдено.

- `410 Gone` — соревнование уже было помечено как удалённое ранее (`ItDeleted == true`).

## Results API (`/api/results`)

### GET `/api/results/`

Получить все **активные** результаты (`ItDeleted == false`).

- Тело запроса: отсутствует.

**Ответы:**

- `200 OK` — есть хотя бы один активный результат.  
  Пример ответа:
  ```json
  [
    {
      "id": 1,
  ```

***

- `204 No Content` — если для данного соревнования нет ни одного активного результата.

### POST `/api/results/`

Создать новый результат.

- Тело запроса (JSON), модель `Result`:

```json
{
  "id": 3,
  "competitionId": 1,
  "participantName": "Сергей Сергеев",
  "place": 3,
  "score": 115.50,
  "itDeleted": false
}
```

**Валидация:**

- `ParticipantName` не пустой (`!string.IsNullOrWhiteSpace`).
- `Place > 0`.
- `CompetitionId > 0`.
- Существует соревнование с данным `CompetitionId` в `competitions`.
- В списке `results` ещё нет результата с таким же `Id`.

**Ответы:**

- `201 Created` — результат успешно создан.  
  Заголовок `Location: /api/results/{id}`.  
  Пример ответа:
  ```json
  {
    "id": 3,
    "competitionId": 1,
    "participantName": "Сергей Сергеев",
    "place": 3,
    "score": 115.50,
    "itDeleted": false
  }
  ```

- `400 Bad Request` — при нарушении любой из проверок.  
  Возможные тексты ответов:
  ```json
  "ParticipantName не должен быть пустым."
  ```
  ```json
  "Place должен быть больше 0."
  ```
  ```json
  "CompetitionId должен быть больше 0."
  ```
  ```json
  "Соревнование с таким CompetitionId не существует."
  ```
  ```json
  "Результат с таким ID уже существует."
  ```

***

### PUT `/api/results/{id}`

Обновить существующий результат.

- Параметры пути:
  - `id` (int) — идентификатор результата.
- Тело запроса (JSON), модель `Result`:

```json
{
  "competitionId": 1,
  "participantName": "Иван Иванов",
  "place": 1,
  "score": 130.00,
  "itDeleted": false
}
```

**Логика/валидация:**

- В списке `results` ищется индекс результата с данным `id`. Если не найден — 404.
- Проверяется:
  - `ParticipantName` не пустой.
  - `Place > 0`.
  - В списке `competitions` существует соревнование с `updatedResult.CompetitionId`.
- При любой ошибке — 400 Bad Request (без текстового сообщения).
- При успехе создаётся новый `Result` с:
  - `Id = id` из пути.
  - Остальные поля — из `updatedResult`.
  - Список сохраняется в файл и возвращается обновлённый объект.

**Ответы:**

- `200 OK` — результат успешно обновлён.  
  Пример ответа:
  ```json
  {
    "id": 1,
    "competitionId": 1,
    "participantName": "Иван Иванов",
    "place": 1,
    "score": 130.00,
    "itDeleted": false
  }
  ```

- `400 Bad Request` — если:
  - `ParticipantName` пустой/пробельный,  
  - или `Place <= 0`,  
  - или не существует соревнования с указанным `CompetitionId`.  

- `404 Not Found` — если результат с данным `id` не найден.

***
