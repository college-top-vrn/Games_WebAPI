# Методы API

## Соревнования
***
### `GET /api/competitions`
#### Описание
Метод, позволяющий получить список всех соревнований в формате JSON.
#### Ожидаемые коды
- `200` — успешный запрос.
***
### `GET /api/competitions/{id}`
#### Описание
Метод, позволяющий получать конкретное соревнование по его идентификатору в формате JSON.
#### Ожидаемые коды
- `200` — успешный запрос;
- `404` — запрашиваемый ресурс не найден.
***
### `POST /api/competitions`
#### Описание
Метод, позволяющий добавлять новое соревнование. Если одно из полей не имеет значение, то создание прерывается с ошибкой.
#### Тело запроса
```json
{
  "Name": "<значение>",
  "Date": "<значение>",
  "Location": "<значение>",
  "SportType": "<значение>"
}
```
#### Ожидаемые коды
- `201` — ресурс успешно создан;
- `400` — неверные входные данные.
***
### `PUT /api/competitions/{id}`
#### Описание
Метод, позволяющий обновлять конкретное существующее соревнование по его идентификатору. Те поля, которые не заполнены, не будут менять значение
#### Тело запроса
```json
{
  "Name": "<значение>",
  "Date": "<значение>",
  "Location": "<значение>",
  "SportType": "<значение>"
}
```
#### Ожидаемые коды
- `201` — ресурс успешно создан;
- `404` — запрашиваемый ресурс не найден.
***
### `DELETE /api/competitions/{id}`
#### Описание
Метод, позволяющий мягко удалять конкретное существующее соревнование по его идентификатору
#### Ожидаемые коды
- `200` — успешный запрос;
- `404` — запрашиваемый ресурс не найден.
***
